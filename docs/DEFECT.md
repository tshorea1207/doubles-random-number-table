## 詳細

### D01: 初回ラウンドで番号の小さいプレイヤーが休憩になる

- **発見日**: 2026-02-09
- **重要度**: 高
- **状況**: 修正済み

**再現手順**:

1. 2コート、12人、1ラウンドで生成
2. 対戦プレイヤーが 5,6,7,8 / 9,10,11,12 となる

**期待動作**:

- 対戦プレイヤーが 1,2,3,4 / 5,6,7,8 となり、9,10,11,12 が休憩

**原因**:

`createFirstRound()` で `generateRestingCandidates` を呼ぶ際、初期 restCounts が全て 0 のため全探索モードに入り、`generateCombinations` が辞書順で `[1,2,3,4]` を最初に返す。結果、番号の小さいプレイヤーが休憩になる。

**対象ファイル**: `src/hooks/useScheduleGenerator.ts` - `createFirstRound()`
